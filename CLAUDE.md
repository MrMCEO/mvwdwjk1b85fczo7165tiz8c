# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Проект

BioWars — Telegram-бот (не web-app) с механикой био-войн. Игроки качают вирусы для атаки и иммунитет для защиты, добывают/покупают ресурсы, заражают других игроков.

## Стек

- **Python 3.12+**
- **aiogram 3.x** (последняя версия)
- **SQLAlchemy 2.x** + **aiosqlite** (асинхронная работа с SQLite)
- **Alembic** — миграции БД
- **Pydantic** — валидация и конфигурация (.env)

## Команды

```bash
# Запуск бота
python -m bot.main

# Миграции
alembic revision --autogenerate -m "описание"
alembic upgrade head

# Тесты
pytest
pytest tests/test_file.py::test_name   # один тест
pytest -x                              # остановка на первой ошибке

# Линтинг
ruff check .
ruff format .
```

## Архитектура

```
bot/
├── main.py              # Точка входа, создание Bot + Dispatcher
├── config.py            # Настройки из .env через Pydantic
├── handlers/            # Роутеры aiogram — по 1 файлу на фичу
├── keyboards/           # Inline и reply клавиатуры
├── models/              # SQLAlchemy-модели (User, Virus, Immunity, Resource)
├── services/            # Бизнес-логика игры (атака, защита, ресурсы, прокачка)
├── middlewares/          # DB-сессия, throttling
└── utils/               # Хелперы
alembic/                 # Миграции БД
tests/                   # pytest-тесты
```

**Поток данных**: Handler → Service → Model (БД). Хендлеры не содержат бизнес-логику — только принимают ввод и вызывают сервисы.

## Игровые механики

- **Атака**: прокачка вируса → заражение других игроков → получение ресурсов от заражённых
- **Защита**: прокачка иммунитета с ветками (разные билды защиты)
- **Ресурсы**: добыча, донат, пассивный доход от заражённых
- **Донат**: универсальная валюта, конвертируемая в ресурсы для атаки или защиты

## Рабочий процесс агентов

**КРИТИЧЕСКОЕ ПРАВИЛО: Opus НИКОГДА не пишет код сам, если пользователь явно не указал это сделать.** Вся работа делегируется агентам.

### Агенты (создаются в начале каждого сеанса)

Постоянные агенты на **Sonnet** (работают всю сессию):

| Агент | Роль |
|-------|------|
| **frontend** | Клавиатуры, тексты сообщений, UX бота |
| **backend** | Хендлеры, сервисы, модели, бизнес-логика |
| **tester** | Пишет и запускает тесты |
| **reviewer** | Код-ревью после каждого изменения |
| **bug-hunter** | Поиск багов, edge-кейсов, уязвимостей |
| **architect** | Планирование структуры, схема БД, связи модулей |
| **documenter** | Обновление документации и комментариев |
| **db-specialist** | Ревью миграций, оптимизация запросов, схема БД |

Финальный агент на **Haiku**:
| Агент | Роль |
|-------|------|
| **committer** | Коммит и пуш на GitHub после завершения работы всех агентов |

### Правила

- Если ни один существующий агент не может решить задачу — Opus создаёт нового
- Агенты работают параллельно где возможно
- Reviewer проверяет код ПОСЛЕ написания, не до
- Committer запускается ПОСЛЕДНИМ
