# BioWars Balance Redesign v1.0

**Дата:** 2026-04-07
**Статус:** Утверждено после 3 раундов обсуждения 10 агентов (7 основных + 3 специализированных)
**Ветка:** dev

## Контекст проекта

BioWars — Telegram-бот с механикой био-войн. Игроки качают вирусы (3 ветки: LETHALITY, CONTAGION, STEALTH) и иммунитет (3 ветки: BARRIER, DETECTION, REGENERATION), добывают ресурсы, атакуют других игроков. Инфекции пассивно дрейнят монеты жертвы.

---

## Критические баги в текущей системе

### 1. Три ветки НЕ РАБОТАЮТ в бою

| Ветка | Что должна делать | Что делает сейчас |
|-------|------------------|-------------------|
| LETHALITY | Увеличивать drain/tick | `damage_per_tick` записывается в `combat.py:342`, но `tick.py:140` его **ИГНОРИРУЕТ** — дрейн = `max(10, victim_total_level * 2)` |
| CONTAGION | Повышать шанс атаки | `effect_value` записывается в `upgrade.py:312`, но **НЕ ИСПОЛЬЗУЕТСЯ** в формуле шанса `combat.py:270` |
| BARRIER | Снижать входящий урон | `effect_value` записывается в `upgrade.py:312`, но **НЕ ИСПОЛЬЗУЕТСЯ** в защитной формуле |

**Файлы:** `bot/services/combat.py`, `bot/services/tick.py`, `bot/services/upgrade.py`

### 2. Экспоненциальная стоимость улучшений

Формула `int(80 * 1.25^level)` в `bot/services/upgrade.py:82-94`:
- Уровень 30: 64,623 монет за 1 апгрейд
- Уровень 50: 6,995,316 монет
- Всего на макс (6 веток по 50): ~140М монет
- При доходе ~3,050/ч на максе = **1,649 дней** до макса

### 3. Новичок теряет 60% баланса за 1 атаку

`combat.py:311-323`: `reward = int(avg_level * 50 * risk_multiplier)`
- Уровень 20 атакует новичка: награда = 300 монет (60% от стартовых 500)
- После инфекции: ещё ~125 монет за 12 часов дрейна

### 4. Антидот смешно дёшев для богатых

`combat.py:523`: `cure_cost = ceil(damage_per_tick * 8)` — фиксированная цена.
- Игрок с 100,000 монет платит 800 за лечение (0.8% баланса)
- Один тик инфекции забирает больше 800 → лечиться ВСЕГДА выгоднее → инфекции бессмысленны для богатых

## НОВЫЕ ФОРМУЛЫ

### 1. Стоимость улучшений

**Было:** `int(base_cost * 1.25^level)` (экспоненциальная)
**Стало:** `300 * L + 8 * L²` (квадратичная)

```python
def calc_upgrade_cost(base_cost, multiplier, current_level):
    L = current_level + 1
    return 300 * L + 8 * L * L
```

| Уровень | Старая | Новая | Разница |
|---------|--------|-------|---------|
| 1 | 80 | 308 | +228 |
| 5 | 195 | 1,700 | +1,505 |
| 10 | 596 | 3,800 | +3,204 |
| 20 | 5,551 | 9,200 | +3,649 |
| 30 | 64,623 | 16,200 | **-48,423** |
| 50 | 6,995,316 | 35,000 | **-6,960,316** |

Всего на макс (6 веток по 50): **~3.8М** (было 140М)

### 2. Добыча (майнинг)

**Было:** `50 + total_level * 10`
**Стало:** `max(50, 120 - 4 * total_level)`

Добыча УБЫВАЕТ с уровнем — заставляет высокоуровневых PvPшить.

```python
base_reward = max(50, 120 - 4 * total_level)
```

| total_level | Старая | Новая |
|-------------|--------|-------|
| 0 | 50 | 120 |
| 5 | 100 | 100 |
| 10 | 150 | 80 |
| 18+ | 230+ | 50 (пол) |

**Принцип:** Уровни 1-5 = можно жить на добыче. Уровень 10+ = нужны инфекции. Уровень 18+ = добыча только подспорье.

### 3. Инфекционный доход


**Было:** `200 + total_level * 20`, стрик 7 дней, макс 1.90x

## БОЕВАЯ СИСТЕМА

### 1. Шанс атаки (экспоненциальный, с CONTAGION и BARRIER)

**Было:** `0.50 + (virus_level - immunity_level) * 0.007` (линейный, ветки НЕ используются)
**Стало:**

```python
import math

# CONTAGION теперь НЕ влияет на шанс (см. раздел "Contagion rework")
base_chance = 0.35 + 0.55 * (1 - math.exp(-virus_level * 0.08))
barrier_penalty = barrier_level * 0.025
chance = base_chance - barrier_penalty
chance = max(0.05, min(0.90, chance))
```

| virus_level | barrier=0 | barrier=10 | barrier=20 |
|-------------|-----------|------------|------------|
| 5 | 54% | 29% | 5% |
| 10 | 68% | 43% | 18% |
| 20 | 80% | 55% | 30% |
| 30 | 85% | 60% | 35% |
| 50 | 89% | 64% | 39% |

### 2. Contagion — ПЕРЕРАБОТАН

**Было:** Множитель шанса атаки (хранился, но НЕ ИСПОЛЬЗОВАЛСЯ)
**Стало:** Определяет **ДЛИТЕЛЬНОСТЬ инфекции**

```python
infection_duration_ticks = 6 + contagion_level * 2
```

| contagion_level | Длительность |
|-----------------|-------------|
| 0 | 6 тиков (6 часов) |
| 5 | 16 тиков |
| 10 | 26 тиков |
| 20 | 46 тиков |

Вирус = шанс попасть. Contagion = как долго длится drain.

### 3. Tick drain (использует LETHALITY и BARRIER)

**Было:** `max(10, victim_total_level * 2)` (ИГНОРИРУЕТ lethality и barrier)
**Стало:**

```python
# damage_per_tick записывается при создании инфекции:
damage_per_tick = max(1.0, 5.0 + lethality_level * 2.0)

# Barrier снижает drain (кап 70%):
barrier_effect = barrier_level * 1.5 * (1 - barrier_level / 120)  # мягкое масштабирование
actual_drain = max(math.floor(damage_per_tick * 0.30), damage_per_tick - barrier_effect)

# Drain не ниже 30% от damage_per_tick (70% кап на снижение)
```

| LETHALITY | damage | BARRIER=0 | BARRIER=10 | BARRIER=20 |
|-----------|--------|-----------|------------|------------|
| 0 | 5 | 5 | 2 (30% пол) | 2 (30% пол) |
| 10 | 25 | 25 | 12 | 8 (30% пол) |
| 20 | 45 | 45 | 32 | 19 |
| 30 | 65 | 65 | 52 | 39 |

### 4. Auto-cure

**Было:** `0.05 + 0.03 + regen * 0.02`, кап 80%
**Стало:**

```python
cure_chance = regen_level * 0.025
cure_chance = min(0.60, cure_chance)  # кап 60% (было 80%)
```

### 5. Награда за атаку (асимметричный лут)

**Было:** `int(avg_level * 50 * risk_multiplier)` — привязано к уровню
**Стало:** Привязано к ДОХОДУ жертвы + направление атаки

```python
victim_hourly_income = max(50, 120 - 4 * victim_total_level) + infection_passive_income
base_reward = victim_hourly_income * 3  # ~3 часа дохода жертвы

# Асимметричный лут-модификатор:
level_diff = attacker_total_level - victim_total_level
if level_diff > 0:  # атака ВНИЗ — штраф
    loot_mult = max(0.05, 1.0 - level_diff * 0.04)
else:  # атака ВВЕРХ — бонус
    loot_mult = min(1.5, 1.0 + abs(level_diff) * 0.02)

reward = int(base_reward * loot_mult)
```

| Ситуация | level_diff | loot_mult | Пример |
|----------|-----------|-----------|--------|
| Равные | 0 | 1.0x | 150 (при доходе 50/ч) |
| +10 вниз | +10 | 0.60x | 90 |
| +25 вниз | +25 | 0.05x (пол) | 7 |
| -10 вверх | -10 | 1.20x | 180 |
| -25 вверх | -25 | 1.50x (кап) | 225 |

### 6. Масштабирование цен (антидот, лаба, крафт)

**Было:** Фиксированные цены (`ceil(damage_per_tick * 8)`)
**Стало:** Масштабируются по уровню

```python
power_score = total_level * 150  # ТОЛЬКО уровень, без баланса (невозможно манипулировать)
cost_multiplier = max(1.0, min(8.0, power_score / 600))  # кап 8.0x

cure_cost = math.ceil(damage_per_tick * 8 * cost_multiplier)
# Все цены лаборатории: base_price * cost_multiplier
```

| total_level | power_score | multiplier | cure (dmg=25) | cure (dmg=65) |
|-------------|-------------|------------|---------------|---------------|
| 3 | 450 | 1.0 | 200 | 520 |
| 10 | 1,500 | 2.5 | 500 | 1,300 |
| 20 | 3,000 | 5.0 | 1,000 | 2,600 |
| 50 | 7,500 | 8.0 (кап) | 1,600 | 4,160 |

**Донат (premium) НЕ масштабируется** — фиксированный курс для всех.
