## Фаза 1: Модели данных (2-3 дня)

### 1.1 Пользователь
```
User:
  - tg_id: BigInteger (PK)
  - username: String
  - bio_coins: Integer (основная валюта)
  - premium_coins: Integer (донат-валюта)
  - created_at: DateTime
  - last_active: DateTime
```

### 1.2 Вирус (атака)
```
Virus:
  - id: Integer (PK)
  - owner_id: FK → User
  - name: String
  - level: Integer
  - attack_power: Integer        # сила заражения
  - spread_rate: Float           # шанс распространения (0.0–1.0)
  - mutation_points: Integer     # очки для веток прокачки
```

### 1.3 Ветки прокачки вируса
```
VirusUpgrade:
  - id: Integer (PK)
  - virus_id: FK → Virus
  - branch: Enum (LETHALITY, CONTAGION, STEALTH)
  - level: Integer
  - effect_value: Float
```

**Ветки вируса:**
- **Летальность** — больше урона заражённым (выкачивает больше ресурсов)
- **Заразность** — выше шанс передачи другим игрокам
- **Скрытность** — сложнее обнаружить и вылечить

### 1.4 Иммунитет (защита)
```
Immunity:
  - id: Integer (PK)
  - owner_id: FK → User
  - level: Integer
  - resistance: Integer          # общая сопротивляемость
  - detection_power: Float       # шанс обнаружить скрытный вирус
  - recovery_speed: Float        # скорость излечения
```

### 1.5 Ветки прокачки иммунитета
```
ImmunityUpgrade:
  - id: Integer (PK)
  - immunity_id: FK → Immunity
  - branch: Enum (BARRIER, DETECTION, REGENERATION)
  - level: Integer
  - effect_value: Float
```

**Ветки иммунитета:**
- **Барьер** — снижает шанс заражения
- **Детекция** — обнаруживает скрытные вирусы, показывает кто атаковал
- **Регенерация** — быстрее избавляешься от заражения, меньше потерь ресурсов

### 1.6 Заражения
```
Infection:
  - id: Integer (PK)
  - attacker_id: FK → User       # чей вирус
  - victim_id: FK → User         # кто заражён
  - started_at: DateTime
  - damage_per_tick: Float       # сколько ресурсов утекает за тик
  - is_active: Boolean
```

### 1.7 Ресурсы и транзакции
```
ResourceTransaction:
  - id: Integer (PK)
  - user_id: FK → User
  - amount: Integer
  - currency: Enum (BIO_COINS, PREMIUM_COINS)
  - reason: Enum (MINING, INFECTION_INCOME, INFECTION_LOSS, UPGRADE, DONATION, DAILY_BONUS)
  - created_at: DateTime
```

## Фаза 2: Базовая экономика (2-3 дня)

### 2.1 Добыча ресурсов
- Команда/кнопка "Добыть" — кулдаун 1 час, даёт bio_coins (случайное количество)
- Ежедневный бонус — заход раз в 24ч даёт фиксированную сумму
- Стрик бонус — последовательные дни увеличивают награду

### 2.2 Система прокачки
- Сервис `bot/services/upgrade.py`
- Формула стоимости: `base_cost * (multiplier ^ current_level)`
- Каждая ветка качается независимо
- Отображение текущих статов и стоимости следующего уровня

### 2.3 Донат-система
- Premium coins покупаются (заглушка, интеграция платежей позже)
- Конвертация premium → bio по курсу
- Premium coins ускоряют кулдауны, дают cosmetic-бонусы
- **Принцип**: донат даёт удобство и ускорение, НЕ абсолютное преимущество

## Фаза 3: Боевая система (3-4 дня)

### 3.1 Атака (заражение)
- Команда "Атаковать" → выбор цели (по username или из списка)
- Расчёт: `attack_power * spread_rate` vs `resistance * (1 + barrier_bonus)`
- Если атака успешна → создаётся Infection
- Кулдаун на атаку (например, 30 минут)
- Уведомление жертве о заражении

### 3.2 Пассивный урон
- Фоновая задача (aiogram scheduler или apscheduler)
- Каждый тик (например, 1 час): заражённые теряют bio_coins → атакующий получает часть
- Размер потерь зависит от ветки "Летальность" атакующего
- Размер потерь снижается веткой "Регенерация" жертвы

### 3.3 Излечение
- Автоматическое: шанс вылечиться каждый тик (зависит от recovery_speed)
- Ручное: потратить ресурсы на мгновенное излечение
- Ветка "Детекция": показывает кто атаковал (без неё — анонимно)

### 3.4 Защита от атаки
- Расчёт при атаке учитывает все ветки иммунитета
- Ветка "Барьер" — прямое снижение шанса заражения
- Ветка "Детекция" vs "Скрытность" вируса — определяет видит ли жертва атакующего

## Фаза 4: Интерфейс бота (2-3 дня)

### 4.1 Главное меню
```
🧬 BioWars
├── 🦠 Мой вирус      → статы, прокачка, ветки
├── 🛡 Мой иммунитет   → статы, прокачка, ветки
├── ⚔️ Атаковать       → выбор цели, атака
├── 💰 Ресурсы         → баланс, добыча, конвертация
├── 🏆 Рейтинг        → топ по заражениям, уровню, богатству
├── 📊 Мой профиль     → общая инфо, активные заражения
└── 💎 Магазин         → донат, обмен валют
```

### 4.2 Inline-клавиатуры
- Навигация по меню через callback_query
- Пагинация для списков (рейтинг, список заражений)
- Подтверждение перед дорогими действиями (прокачка, атака)

### 4.3 Уведомления
- Тебя атаковали / заразили
- Ты вылечился
- Пассивный доход получен
- Кулдаун закончился (опционально)

## Фаза 6: Балансировка и тестирование (2-3 дня)

### 6.1 Балансировка
- Формулы урона, защиты, прокачки — подобрать коэффициенты
- Убедиться что все ветки жизнеспособны (нет одного доминирующего билда)
- Экономика: приход ресурсов ≈ расход, без гиперинфляции
- Донат не делает игрока неуязвимым

### 6.2 Тестирование
- Unit-тесты сервисов (атака, прокачка, ресурсы)
- Интеграционные тесты (хендлер → сервис → БД)
- Тест граничных случаев (0 ресурсов, максимальный уровень, атака самого себя)

## Фаза 7: Дополнительные механики (после MVP)

Идеи для развития после запуска базовой версии:

### 7.1 Мутации вируса
- Рандомные события: вирус мутирует, получает временный бафф/дебафф
- Редкие мутации дают уникальные способности

### 7.2 Альянсы / кланы
- Объединение игроков в группы
- Совместные атаки на сильных игроков
- Клановый иммунитет (общий бонус к защите)

### 7.3 Эпидемии (ивенты)
- Временные события на сервере
- Босс-вирус: все объединяются для защиты
- Усиленная добыча ресурсов на время ивента

### 7.4 Лаборатория
- Крафт предметов из ресурсов
- Вакцины (одноразовая защита от конкретного вируса)
- Биооружие (усиленная одноразовая атака)

### 7.5 Чёрный рынок
- Торговля ресурсами между игроками
- Аукцион редких мутаций
- Контракты на заражение (заказные атаки)
